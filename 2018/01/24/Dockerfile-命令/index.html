Dockerfile 命令

发表于 2018-01-24
字数统计 990 | 阅读时长 4

Dockerfile是使用Docker必须掌握的，它用于快速创建Docker, Dockerfile文件的注释都是“#”开始的，每一行都是一个命令，Dockerfile命令分为4个部分：基础镜像、维护者信息、镜像操作指令和容器启动指令。
例子：

FROM relateiq/oracle-java8:latest
COPY target/spring-test.jar /test/spring-test.jar
WORKDIR /test
EXPOSE 9090
CMD ["/bin/sh", "-c", "java -jar ./spring-test.jar"]

FROM

FROM \<image> [AS \<name>]

FROM \<image>[:\<tag>] [AS \<name>]

FROM \<image>[@\<digest>] [AS \<name>]

FROM 指令会在基础镜像上创建新的镜像，这条指令了会去公共的仓库拉取镜像，可以和ARG 指令(唯一一个可以在FROM 指令前使用的命令)一起使用，tag和digest时可选择的, tag的默认值是latest，如果在构建的时候没有找到标签会构建失败。
例子：

ARG VERSION=latest
FROM base:$VERSION

MAINTAINER

MAINTAINER用来写维护者的信息。

例子:

MAINTAINER BuHuan Huang <buhuanhuang@gmail.com>

ADD

ADD指令用来将构建环境的文件和目录复制到镜像中，如果源文件是压缩文件，会进行解压缩，支持gzip、bzip、xz等。在Linux的容器中可以使用–chown=\:\命令。

例子: 

# 添加所有hom开头的文件
ADD hom* /mydir/       
# 添加hom开头.txt结尾的文件
ADD hom?.txt /mydir/    
# 将target文件夹中的test.jar文件复制到容器的test目录下
ADD target/test.jar /test/test.jar

COPY

和ADD指令相似，不过不会进行文件的提取和解压缩工作，由改指令创建的文件或目录的UID和GID都会这只为0。

例子：

COPY target/test.jar /test/test.jar

RUN

RUN指令用于构建时执行某些命令。

例子：

FROM ubuntu
RUN echo foo > bar

ARG

ARG指令用来定义可以在docker build时传递给构建的变量

ARG USER
ARG VERSION=latest

在使用命令docker build --build-arg USER=hbh -t hbh/test .时会把user的值设置为hbh，因为没有传入VERSION的值，所以VERSION会使用默认值latest。

ENV

ENV指令可以用来设置环境变量, 也可以在启动的时候设置docker run -ti -e "JAVA_HOME=/usr/local/java" test env
例子：

ENV JAVA_HOME /usr/local/java

EXPOSE

EXPOSE指令暴露对外端口

例子：

EXPOSE 8080

WORKDIR

WORKDIR指令可以在创建容器时，在容器内部设置一个工作目录，让ENTRYPOINT或CMD命令的指定程序在这个目录下运行.

例子：

# 在test目录下使用java -jar test.jar命令
WORKDIR /test
CMD ["/bin/sh", "-c", "java -jar test.jar"]

USER

指定容器由什么用户运行，如果不使用user默认使用root用户运行。

例子：

USER user
USER user:group
USER uid
USER uid:gid
USER user:gid
USER uid:group

ONBUILD

ONBUILD指令能为镜像添加触发器，当一个镜像被其他镜像用作基础镜像时，如果不想将某条命令在基础镜像执行而在其他镜像执行时使用。

例子：

ONBUILD ADD . /app/src
ONBUILD cd /app/src && make

VOLUME

VOLUME 指令可以指定某些目录挂载文为匿名卷，这样运行的时候不会向容器写入大量的数据，并可以使用数组挂载多个目录。

例子：

VOLUME ["/opt"]
VOLUME ["/opt", "/data"]

LABEL

LABEL指令用于为Docker镜像添加元数据, 推荐将所有的元数据都放到一条LABEL标签中，以防止创建过多的镜像层。

例子：

LABEL version="1.0"
LABEL user1="zs" user2 ="lisi"

CMD

CMD指令在容器启动的时候执行，由于/bin/bash -c 在执行命令的时候会导致意料之外的行为，所以官方推荐使用以数组来设置要执行的命令，CMD命令在容器启动的时候可以被覆盖如 docker run - i -t hbh/test /bin/ps 这样就不会运行指定的CMD命令了。

例子：

CMD ["executable", "param1", "param2"]
CMD ["/bin/sh", "-c", "java -jar test.jar"]

ENTRYPOINT

ENTRYPOINT指令与CMD指令相似，不过ENTRYPOINT指令不会在容器启动的时候被覆盖

ENTRYPOINT ["executable", "param1", "param2"]
